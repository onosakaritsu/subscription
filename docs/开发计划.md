# 开发计划

## 当前目标

项目主线为本地优先的订阅管理 Web App。保持 Node.js 原生 HTTP 后端、静态 HTML/CSS/JS 前端、本地 JSON 存储，不使用 Apple Development Team、App Group、WidgetKit、数据库或第三方框架。

## Web V1 任务清单

- [x] 评估无 Apple Development Team 条件下的技术栈。
- [x] 选定零依赖 Node.js 原生 HTTP + 静态前端 SPA + 本地 JSON。
- [x] 取消小组件需求，不再实现 Widget 风格页面或 Widget API。
- [x] 将项目主名改为 `Project_03_subscription-manager-web`。
- [x] 实现订阅领域逻辑：分类、币种、周期、续费日计算、管理排序、统计。
- [x] 实现 HTTP API：增删改查、导入、导出、健康检查。
- [x] 实现前端主界面：列表、表单、搜索、筛选、统计。
- [x] 使用 `data/subscriptions.json` 本地持久化。
- [x] 增加 Node.js 单元测试。
- [x] 清理 SwiftUI / WidgetKit / XcodeGen 遗留文件。
- [x] 同步 README、开发计划、踩坑日志。

## Phase 2：桌面化使用与订阅状态增强

- [x] 保持 V1 基线功能：增删改、自动/手动续费日、筛选、统计、导入导出、健康检查、页面访问。
- [x] 新增续费状态标签：已过期、今日续费、3 日内、7 日内、本月内、正常、已停用。
- [x] 状态计算放入领域层，可复用并有测试覆盖。
- [x] 优化列表默认排序：启用优先、过期最前、停用置底。
- [x] 首页统计卡片：本月需续费、7 日内续费、启用、停用、月度折算、年度折算。
- [x] 多币种分别统计，不做自动汇率换算。
- [x] 新增半年付和一次性周期；一次性不混入周期性折算。
- [x] 增强最近续费摘要，最多显示 5 条紧急续费项目。
- [x] 增加本地自动备份：`data/backups/`，最多保留 20 份。
- [x] 优化 JSON 导出文件名：`subscriptions-backup-YYYY-MM-DD.json`。
- [x] 新增 `start-subscription-manager.sh` 一键启动脚本并设置可执行权限。
- [x] 优化 420px～600px 窄窗口桌面化使用体验。
- [x] 默认 `README.md` 改为中文版，新增 `README.en.md`。
- [x] 补充 Phase 2 测试。
