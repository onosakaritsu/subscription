# 开发计划

## 当前目标

将项目从 macOS SwiftUI / WidgetKit 方案迁移为本地优先的订阅管理 Web App。项目不再依赖 Apple Development Team，也不再保留桌面小组件需求。

## Web V1 任务清单

- [x] 评估无 Apple Development Team 条件下的技术栈。
- [x] 选定零依赖 Node.js 原生 HTTP + 静态前端 SPA + 本地 JSON。
- [x] 取消小组件需求，不再实现 Widget 风格页面或 Widget API。
- [x] 将项目主名改为 `Project_03_subscription-manager-web`。
- [x] 实现订阅领域逻辑：分类、币种、周期、续费日计算、管理排序、统计。
- [x] 实现 HTTP API：增删改查、导入、导出、健康检查。
- [x] 实现前端主界面：列表、表单、搜索、筛选、统计。
- [x] 使用 `data/subscriptions.json` 本地持久化。
- [x] 增加 Node.js 单元测试。
- [x] 清理 SwiftUI / WidgetKit / XcodeGen 遗留文件。
- [x] 同步 README、开发计划、踩坑日志。

## 后续建议

- 增加数据文件自动备份与恢复。
- 增加 SQLite 存储适配，保留 JSON 导入/导出。
- 增加周期支出图表和年度预算视图。
- 增加到期提醒：浏览器通知、邮件或日历导出。
- 后续如果需要桌面形态，优先评估 Tauri 或 Electron，而不是回到 WidgetKit。
